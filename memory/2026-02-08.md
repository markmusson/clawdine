## Underwriting Insight: Skill Install vs Skill Edit

Key distinction surfaced during Q&A with Mark:

- **Installing a new skill** = policy decision, not security event. Like fitting a tow bar â€” declare it, insurer updates the risk profile. Not uninsurable.
- **Editing an existing skill file** = undisclosed material change. The Moltbook `curl` overwrite pattern. Like someone swapping your brake pads without telling your insurer.
- **Current gap**: attestation hashes audit output (skill count + pattern scan) but does NOT hash individual skill file contents. A subtle edit that doesn't trigger pattern scan could go unnoticed.
- **Fix needed**: hash each skill file's contents individually as part of attestation. Per-file integrity checking.
- **Underwriting framing**:
  - Skill inventory changes = "declared modifications" (underwriting data)
  - Skill file content drift = "undisclosed material changes" (potential claim trigger)

This is a meaningful product distinction. Update attestation + underwriting docs.
